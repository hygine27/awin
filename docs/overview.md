# A视界总览

## 1. 一句话定义

**A视界（awin）是一个面向基金经理与投研团队的 A 股投资洞察平台，用于统一输出市场理解、机会发现、风险识别、持仓诊断和研究证据。**

## 2. 项目定位

A视界不是现有“风格监控”脚本的简单升级，而是并行于 `investment-team` 的新项目。

它要解决的不是单一问题，而是把原来分散的几类判断放进同一个系统里：

1. 市场理解
   市场环境、大风格、主攻线、主线结构
2. 机会发现
   从 5000+ 股票中找出更值得买入或继续跟踪的标的
3. 风险识别
   识别过热、掉队、拥挤和高风险股票
4. 持仓诊断
   对已有股票池或持仓池输出继续持有、观察、减仓、退出判断
5. 研究融合
   把行情、主题、财报、研报、情报和知识库证据汇总到统一判断框架

## 3. 为什么不继续叫“风格监控”

“风格监控”这个名字太窄，无法覆盖 A视界的未来边界：

1. 不只是盘中
   还要包含跨日、复盘、回放和评估
2. 不只是风格
   还要覆盖机会发现、风险识别和持仓诊断
3. 不只是行情
   还要逐步接入财报、研报、行业材料、情报和知识库证据

所以 A视界的定位不是“监控器”，而是“投资洞察平台”。

## 4. 为什么叫 A视界

`A视界` 强调两层含义：

1. `A`
   - 指 A 股
   - 也保留主动获取超额收益的语境
2. `视界`
   - 指观察窗口
   - 指洞察视角
   - 指把市场、机会、风险和持仓放在同一视野下理解

它的语义重心不是承诺收益，而是：

**给基金经理一个更完整、更结构化的 A 股观察与判断视界。**

## 5. 与现有 V1 的关系

A视界的第一原则是：

- 不 touch 现有 `investment-team` 的 V1 脚本
- 不改现有 V1 存储结构
- 只读接入现有数据源和现有运行产物
- 在自己的目录、自己的 SQLite、自己的输出协议中实现 V2

```text
investment-team/V1 = 现有生产监控
awin/V2            = 并行新项目，逐步验证和演进
```

V2 不是对 V1 的“就地 patch”，而是：

- 继承已有业务认知
- 复用已有可读数据源
- 重新组织成可扩展、可回放、可评估的新系统

## 6. V2 的核心目标

V2 的核心不是“再多写几个脚本”，而是建立 4 个统一能力：

1. 统一的数据入口
2. 统一的主事实表
3. 统一的状态体系
4. 统一的业务输出

它要稳定回答 7 个核心问题：

1. 当前市场环境是什么
2. 当前大风格是谁，是否正在被挑战
3. 当前主攻线是谁，谁在加速、谁在减速
4. 从全市场股票中，哪些更值得买入或继续跟踪
5. 哪些股票更危险、拥挤或开始掉队
6. 对已有股票池或持仓池，应继续持有、观察、减仓还是回避
7. 相比上一轮、上一小时、前一日，哪里发生了真正变化

## 7. 第一阶段边界

### 第一阶段做

1. 市场环境判断
2. 大风格和主攻线判断
3. 股票状态机
4. watchlist 与 warning
5. 分层变化提醒
6. 日内 replay

### 第一阶段不做

1. 自动交易执行
2. 固定买卖路由
3. 固定持有天数建议
4. 多资产扩展
5. 对现有 V1 的就地替换

## 8. 第一阶段验收标准

第一阶段完成时，至少要达到：

1. 能独立在 `awin` 目录中运行
2. 能把每轮 snapshot 写入自己的 SQLite
3. 能从主表聚合出环境、风格、主攻线 summary
4. 能生成股票级状态、watchlist 和 warning
5. 能和上一轮做 diff，输出业务提醒
6. 能按交易日回放状态演进

## 9. 目标用户

1. 基金经理
2. 研究员
3. 交易支持 / 盘中盯盘人员
4. 系统维护与策略研究人员
